## Looking Ahead: Terms for Next Week
Research and define the following terms to the best of your ability.

1. RESTful routes:
A RESTful route is a set of standards used to create efficient, reuseable routes. The route provides mapping from HTTP verbs to the controller CRUD actions. A RESTful route depends on the HTTP verb and the URL.

2. JSON:
JSON stands for JavaScript Object Notation is a data interchange format. In other words, JSON is a standard text-based format for representing structured data based using JS object syntax. It is used for transmitting data in web applications.

3. ERB:
I believe ERB stands for "Embedded Ruby". It is a HTML file that has ruby code embedded in it.

4. Params:
Params are key-value pairs that is encoded in a HTTP request. URL params, also known as query strings are a way to pass information into a controller method.

5. API:
I believe API stands for application programming interface, which is a software intermediary. This allows two application to talk to each other. The interface has its own set of definitions and protocols for building and intergrating application software.
